Invesco Cloud Dashboard

1-week sprint of data visualization and UI design to co-create an interactive prototype for a client pitch.
Team: Emma Healy (UI designer), Paul Daly (UX designer), C.I. Reichel, and Jennifer Borchardt
April 2023
Skills
  • UI Design
  • Data Visualization
  • Mid-fidelity Prototyping
  • Presentation
Tools
  • Figma

Overview

I had the opportunity to work on a week-long pitch for Invesco, where we were tasked with creating a concept for a cloud usage dashboard. Our users include the Invesco CFO and his team -- allowing them to view trends and determine the business value of cloud spend to make important financial and technical decisions.

We needed to create a dashboard that was both informative and easy to use regardless of the data-rich requirements. In addition to the high-level dashboard, we created a business unit-specific breakdown of cloud usage for the Tech and Services division of Invesco. Invesco asked us to take inspiration from utility bills with a graphic breakdown explaining cloud spending.


The Ask: An overall cloud spend dashboard and a business-unit specific "utility bill."

The Sprint

To tackle this challenge, I worked closely with a UX designer to create a friendly, visual breakdown of cloud spend. I flexed my data-visualization skills to create an interactive Figma prototype that allows users to easily understand business cloud usage and make informed decisions. Overall, it was an exciting and challenging project, and I'm proud of the work we accomplished in just one week.

Our overall approach was to keep customization and scalability at the forefront of the design. In the context of a week long effort, we wanted to show how Sapient Experience might approach data visualization in a dashboard while hinting at the potential opportunities for new features.

At the end of the sprint, I presented our work to the client, explaining the design principles we followed while stressing the business value of the product and potential for a future partnership.

Final Prototype

The design is card based and scalable to allow for expanded features in future iterations. We want the user to be able to customize the arrangement and visibility of cards as well as the content within each widget. Filters also allow for customization and focus.

We included widgets to visualize trend data and AI predictions for future spend. We want users to be able to scan and comprehend data trends, but also have the option to dive deeper to understand the nitty gritty. Throughout the dashboard, we leverage interactive elements to create a bite-snack-meal approach to a data-heavy product.

The cloud onboarding progress widget illustrates how customizable some of this data might be for the user. If this user is most interested in the current state of completed and in-progress applications, a bar graph showing the totals for each category will be most useful for them. But another user might want to see how these applications have progressed over time, in which case a timeline view of how many applications have been completed in the past six months or year might be more useful to illustrate the pace of the progress being made.

We leveraged hover interactions to make this chart more friendly, informative, and inviting. We want users to get the bigger picture at first glance and then encourage them to look closer to get the details. Adding satisfying interactions entices the user to get more information and explore the interface.

This experience taught me the importance of prioritizing key assumptions and best practices in dashboard design, and how to create a scalable and customizable dashboard that can meet the needs of different audiences.

Key Takeaways

With such a condensed timeline, it was crucial that we leverage key assumptions and best practices in dashboard design. While we had a lot of data to pull from, we had no access to our primary users -- the CFO and his leadership team -- for research. So, we kept things simple, distilled information into easy-to-read charts for trend visualization, and always defaulted to customizability. Each widget is designed to be scalable to different audiences -- filter controls are built in, cards can be rearranged, and data can be visually manipulated to suit different needs. Because of the limits of the sprint, our emphasis was always on what is possible with a great framework rather than building out individual widgets to full functionality.

Overall, this sprint was a great experience for me to use my data visualization and UI design skills while in a fast-paced pitch situation. I enjoyed having much ownership over this project as one of two primary designers, which was especially rewarding when I was trusted to present work to the client. I'm excited to see where this project goes in the future and look forward to continuing to work on projects that challenge me and allow me to grow as a visual designer.
